# اختبار نظام إعادة تعيين كلمة المرور الجديد

## التغييرات التي تمت:

1. **تغيير رمز إعادة التعيين**: من رمز طويل (60 حرف) إلى 6 أرقام
2. **تحديث Validation**: للتأكد من أن الرمز مكون من 6 أرقام فقط
3. **تحديث قاعدة البيانات**: تغيير نوع البيانات لحقل `reset_password_token`

## كيفية الاستخدام:

### 1. طلب رمز إعادة التعيين:
```http
POST {{base_url}}/forgot-password
Content-Type: application/json

{
    "email": "samuaeladel3@gmail.com"
}
```

**الاستجابة المتوقعة:**
```json
{
    "success": true,
    "message": "تم إرسال رمز إعادة التعيين إلى بريدك الإلكتروني"
}
```

### 2. إعادة تعيين كلمة المرور:
```http
POST {{base_url}}/reset-password
Content-Type: application/json

{
    "token": "472271",
    "password": "newpassword123",
    "password_confirmation": "newpassword123"
}
```

**الاستجابة المتوقعة:**
```json
{
    "success": true,
    "message": "تم تغيير كلمة المرور بنجاح"
}
```

## مثال للاختبار:

الآن يمكنك اختبار النظام باستخدام الرمز: **472271**

```json
{
    "token": "472271",
    "password": "newpassword123",
    "password_confirmation": "newpassword123"
}
```

## ملاحظات مهمة:

1. **الرمز صالح لمدة ساعتين فقط**
2. **الرمز مكون من 6 أرقام بالضبط**
3. **يتم حذف الرمز بعد الاستخدام**
4. **كلمة المرور يجب أن تكون 8 أحرف على الأقل**
5. **يجب تأكيد كلمة المرور**

## التحقق من صحة الرمز:

- يجب أن يكون الرمز مكون من 6 أرقام فقط
- يجب أن يكون الرمز غير منتهي الصلاحية
- يجب أن يكون الرمز موجود في قاعدة البيانات
