# نظام إعادة إرسال رمز التحقق من البريد الإلكتروني

## التغييرات التي تمت:

1. **إضافة دالة جديدة**: `resendEmailVerification` في `AuthController`
2. **إضافة route جديد**: `POST /resend-email-verification`
3. **التحقق من حالة البريد الإلكتروني**: منع إعادة الإرسال للحسابات المؤكدة بالفعل

## كيفية الاستخدام:

### السيناريو الكامل:

#### 1. إنشاء حساب جديد:
```http
POST {{base_url}}/register
Content-Type: application/json

{
    "full_name": "مستخدم تجريبي",
    "username": "testuser123",
    "email": "test@example.com",
    "phone": "01111111111",
    "country": "مصر",
    "user_type": "tourist",
    "password": "password123",
    "password_confirmation": "password123"
}
```

**الاستجابة المتوقعة:**
```json
{
    "success": true,
    "message": "تم إنشاء الحساب بنجاح. يرجى التحقق من بريدك الإلكتروني",
    "data": {
        "user_id": 2,
        "email": "test@example.com"
    }
}
```

#### 2. إعادة إرسال رمز التحقق (إذا لم يصل):
```http
POST {{base_url}}/resend-email-verification
Content-Type: application/json

{
    "email": "test@example.com"
}
```

**الاستجابة المتوقعة:**
```json
{
    "success": true,
    "message": "تم إعادة إرسال رمز التحقق إلى بريدك الإلكتروني"
}
```

#### 3. تأكيد البريد الإلكتروني:
```http
POST {{base_url}}/verify-email
Content-Type: application/json

{
    "email": "test@example.com",
    "code": "oWzE2W"
}
```

**الاستجابة المتوقعة:**
```json
{
    "success": true,
    "message": "تم تأكيد البريد الإلكتروني بنجاح",
    "data": {
        "user": {...},
        "token": "2|abc123..."
    }
}
```

## مثال للاختبار:

يمكنك الآن اختبار النظام باستخدام:
- **Email**: `test@example.com`
- **رمز التحقق الجديد**: `oWzE2W`

## الحالات المختلفة:

### 1. إعادة الإرسال لحساب غير مؤكد (نجح):
```json
{
    "success": true,
    "message": "تم إعادة إرسال رمز التحقق إلى بريدك الإلكتروني"
}
```

### 2. إعادة الإرسال لحساب مؤكد بالفعل (فشل):
```json
{
    "success": false,
    "message": "البريد الإلكتروني مؤكد بالفعل"
}
```

### 3. إعادة الإرسال لبريد إلكتروني غير موجود (فشل):
```json
{
    "message": "The selected email is invalid.",
    "errors": {
        "email": ["The selected email is invalid."]
    }
}
```

## الميزات:

1. **سهولة الاستخدام**: المستخدم يحتاج فقط لإدخال البريد الإلكتروني
2. **أمان**: التحقق من وجود البريد الإلكتروني في قاعدة البيانات
3. **منع الإساءة**: عدم السماح بإعادة الإرسال للحسابات المؤكدة
4. **رمز جديد**: يتم إنشاء رمز تحقق جديد في كل مرة
5. **نفس المعمارية**: يتبع نفس نمط باقي الـ APIs

## تدفق العمل:

1. المستخدم ينشئ حساب جديد
2. يتم إرسال رمز التحقق للبريد الإلكتروني
3. إذا لم يصل الرمز، يمكن طلب إعادة الإرسال
4. يتم إنشاء رمز جديد وإرساله
5. المستخدم يدخل الرمز الجديد لتأكيد الحساب

## الأمان:

- **التحقق من وجود البريد**: منع الطلبات العشوائية
- **التحقق من حالة التأكيد**: منع إعادة الإرسال غير الضرورية
- **رمز جديد**: كل طلب ينشئ رمز جديد ويلغي القديم
